# Testing & Analysis Methodology

> Before implementing logic changes, follow the 4-step analysis process.
> Full guide: `.claude/skills/testing-methodology/SKILL.md`

1. **Input Assumptions** — list variables, create I/O matrix
2. **Flow Analysis** — trace current flow, document WHY each step exists
3. **Report** — consolidate findings, propose solution
4. **Implement** — only after steps 1-3

Conflict check: Does the change conflict/duplicate existing logic?
